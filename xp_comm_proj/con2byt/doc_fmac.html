Convert2Byte

Synopsis

Convert data in other numeric data types into bytes. This is necessary if the data is to be volume rendered.

Input Ports

inMesh+Node_Datainput field for conversion

Parameters

None.

Output Ports

out Mesh+Node_Data field with node data converted to bytes

Description

Convert2Byte converts the node data of a field from short ints, ints, floats and doubles into bytes scaled to fit into the 0-255 range of byte data.
new_value= old_value * (255/ (maximum - minimum))
Note: Where the input data has context (eg velocities in m/s) this information is lost. Accuracy will also be lost due to the limited range of the byte. Therefore the output of this module should not really be used as the basis for any accurate calculation.

Input Ports

input_name
input field for the Convert2Byte macro. The lower-level module Convert2ByteNodeData is used to convert to bytes the Node_Data of this input field then recombine it with the Mesh (if present).

Parameters

None.

Output Ports

out
The output field for the Convert2Byte macro. This contains the converted Node_Data combined with the original Mesh (if present).

Utility Macros

Parameter Group, Module, Functional Macro and UI Macro.

Example

Convert2ByteEg. Convert2Byte used to convert the floating point values from bluntfin.fld into byte data so that the data can be rendered by the volume renderer.
