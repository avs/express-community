extract_vector

Synopsis

This macro provides a user interface to the DVextr_vector module in the Visualization library. It allows a single output vector to be selected from a number of input components. For each element of the output vector, the input component and the element of that component are selected.

in Field (link) Input field

Description

This macro simply encapsulates a user interface with the low-level module DVextr_vector. It allows a vector component to be generated by selecting its elements to refer to any input component's elements. The desired output vector length is specified, and then for each element, the required input is selected.

For each element of the output vector, an input component must be selected from the Component option menu, which lists all the components of the input field. Secondly, an element from the chosen input component must be selected, as the input component itself may be a vector.

In this way, elements can be mixed at will to generate the desired output. This macro is suitable for processing components which are not generally handled by most AVS/Express visualization macros, such as 6 DOF data (generated by modules such as Read_SDRC.)

Input Ports

in
Input field containing some Node_Data. Components and elements from this data are then extrapolated to produce a new output component vector.

Parameters

As above

Output Ports

out
Output field containing a single component comprised of the selected elements from the input components.

Utility Macros

The existing low-level DVextr_vector module is used in this User Macro extract_vector. The low-level DVextr_vector module is only available to users of the AVS/Express Developer's Edition.

Example

An example application is not provided.

Files

xp_comm_proj/extr_vec/xvecdmacs.v contains the extract_vector User Macro, the ExtractVector Functional Macro and the ExtractVectorUI UI Macro V definitions.

Other Notes

The extract_vector macro contains no additional source code, so does not require compilation. By default the user inteface will execute in the express process.
