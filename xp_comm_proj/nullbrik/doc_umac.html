null_brick

Synopsis

The user macro null_brick modifies a sub-volume of node data within a uniform volume, by replacing all the values with a single given value. This value may optionally be set to be the null_value.

Input Ports

Name Type Description
&in ilink (Mesh_Unif+Node_Data_Struct) input uniform field

Parameters

The following lists all of the parameters found in the parameter block NullBrickParams, which are accessed by the module NullBrickNodeData and the UI Macro NullBrickUI.

Name Type Description UI Control
xmin int sub-volume lower extent for x-axis Brick Extents: xmin
xmax int sub-volume upper extent for x-axis Brick Extents: xmax
ymin int sub-volume lower extent for y-axis Brick Extents: ymin
ymax int sub-volume upper extent for y-axis Brick Extents: ymax
zmin int sub-volume lower extent for z-axis Brick Extents: zmin
zmax int sub-volume upper extent for z-axis Brick Extents: zmax
nval double sub-volume data (null) value to replace all existing data Null/Brick Value
setnull int (toggle) set null_value to be nval Set Null Flag

Output Ports

Name Type Description
out Mesh_Unif+Node_Data_Struct output field with modified values in the sub-volume

Description

The user macrp null_brick inputs a uniform volume (Mesh_Unif+Node_Data_Struct) and takes a set of parameters to define a sub-volume within, and a replacement data value. All node data within the sub-volume is replaced with the given value, in all components. This allows sub-volumes to be cut away - similar to the excavate_brick module - but doesn't convert the field to an unstructured mesh, thus allowing the volume to be volume rendered.

Additionally, the given value, may be specified as the null_value in the Node_Data components, and the null_flag parameter is set.

Input Ports

&in
Input for uniform node data (Node_Data_Struct), which will have a sub-volume modified as defined by the extents in the parameter block, for all components of all primitive data types.

Parameters

xmin
Sub-volume lower extent for the x-axis.
xmax
Sub-volume upper extent for the x-axis.
ymin
Sub-volume lower extent for the y-axis.
ymax
Sub-volume upper extent for the y-axis.
zmin
Sub-volume lower extent for the z-axis.
zmax
Sub-volume upper extent for the z-axis.
nval
Value (null) to be used to fill all values defined as the sub-volume.
setnull
Sets the nval parameter to be the null_value in each component, and sets the null_flag of the Node_Data.

Output Ports

out
Modified Mesh_Unif+Node Data_Struct output.

Utility Macros

The low-level NullBrickNodeData module is used in the functional macro NullBrick and this User Macro null_brick.

Example

Two example applications, NullBrickEg1 and NullBrickEg2 are provided, which show the null_brick macro in use with the volume_render modules and the isosurface modules respectively.

Files

xp_comm_proj/nullbrik/nbrkdmacs.v contains the null_brick User Macro, the NullBrick Functional Macro, the NullBrickUI UI Macro, and the example application NullBrickEg V definitions.

Other Notes

The NullBrickMacs library inherits its process. As this library contains no procedural code, the process is not important. The low-level NullBrickNodeData module, executes under the process specified in its library, not the process defined in the high-level library.
