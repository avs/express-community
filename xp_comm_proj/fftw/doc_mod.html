FFTw_raw

Synopsis

Perform the ``fastest Fourier transform in the west'' (FFTW) on the data in a uniform field.

  module FFTw_raw <c_src_files="cp_utils/avs_err.c cp_utils/data_field.c",
    hdr_dirs="cp_utils fftw/fftw/src",
    src_file="fftw/FFTw_vxp.c">
    {
      Mesh_Unif+Node_Data &in_field<export=2,NEportLevels={2,0}>;
      Mesh_Unif+Node_Data out_field<export=2,NEportLevels={0,2}>;
      FFTw_params &FFTw_params;
      boolean direction => FFTw_params.direction;
      int rescale => FFTw_params.rescale;
      omethod+notify_inst+req FFTupdate
        (direction+read+notify+req,
         in_field+read+notify+req,
         rescale+read+notify+req,
         out_field+write ) = "FFTw_vxp";
    };

Description

FFTw_raw performs the ``fastest Fourier transform in the west'' (FFTW) on the data in a uniform field. FFTW is usually faster (and sometimes much faster) than all other public-domain Fourier Transform programs. The module works on Node_Data of any type or dimension. The dimensions do not have to be powers of two. The output field is a real (double), complex (vector length = 2) uniform field with the same dimensions as the input field.

Inputs

&in_field

A reference to a uniform field with Node_Data of any type.

&FFTw_params

A reference to the parameter group that contains real instances of the direction and rescale values.

direction

A reference to the direction value of type boolean, via the parameter group that is referenced by the &FFTw_params input.

rescale

A reference to the rescale value of type integer, via the parameter group that is referenced by the &FFTw_params input.

Outputs

out_field

A real (double), complex (vector length = 2) uniform field with the same dimensions as the input field.

Utility Macros

The low-level FFTw_raw module is used in the Functional Macro FFT and the User Macro FFTw. Additionally, these macros also use the parameter group FFTw_params, as does the UI Macro FFTw_UI.

Example

An example application FFTwEg is provided that uses the AVS/Express sample data file hydrogen.fld and performs a forward and inverse FFT and displays the result.

Files

xp_comm_proj/fftw/FFTw_mods.v contains the FFTw_params group and the FFTw_raw module V definitions.

xp_comm_proj/cp_utils contains source code for functions called by ntats_raw.

Prerequisites

This project relies on the successful installation of the following components. Without them it will not function correctly.
  • FFTW library
  • cp_utils project

Other Notes

By default the low-level library FFTwMods, which needs to be compiled, has the process set to express.

This module uses functions in the cp_utils library (available from IAC). It must be installed before compiling this project.

Also, before compiling this project you must build the FFTW library (available from http://www.fftw.org). Once FFTW is built it will contain a directory called src that contains header files and a library file (libfftw.a) that are needed to build the FFTw project. Create a link called fftw in the project directory xp_comm_proj/fftw/. This link should point to the FFTW directory that contains src. Once the link is created you can compile the FFTw project using the VCP command given above.
